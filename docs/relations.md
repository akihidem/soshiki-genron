# 関係図 / relations — この研究が束ねる実装群

> soshiki-genron は単独で生まれた問いではない。operator が別々に作ってきた
> **「AI を外部の決定的な錨で検証し、律する」実装群**があり、本研究はそれらを
> *組織論として後ろから理論化する受け皿*である。各実装は本研究の **データ点**であって、
> 本研究はどれも**起点にしない**（機能から再導出する — [`method.md`](method.md)）。
>
> ⚠️ ここに挙げる姉妹リポの多くは operator の private リポ。名前と位置づけの記載は

## 1. 起点 — tehai（手配）→ 問い → 本研究

```
   tehai（手配・private / tehai-core public）        soshiki-genron（本研究・public）
   AI 委譲レイヤの実装                               「AI にとって組織とは何か」を機能から再導出
   request → 契約付き task DAG → registry agents      ③ 概念 → ① 組織論へ転移 → ② 実験基盤(反証可能化)
   → multi-factor model routing → risk review         市場支配定理 p*=w/s・統治膜 m*・レース外部性
   → Judge → sandbox 実行 / Python stdlib only         Goodhart・監督スケーリング・cross-vendor mesh
        │                                                        ▲
        │   「人間組織の形(PM/Arch/役割/階層)を借りて実装した」      │  tehai を起点にせず、
        └──────────  借り物は AI に最適か? という問いが立つ  ──────▶  組織そのものから問い直す
                                              （tehai の A/B 実測は "参考データ点 1 つ" として引用・数値は本リポに転記）
```

`tehai` は人間組織の形を**借りて**作った委譲レイヤ。動かしてみて「この借り物は AI には
擬人的負債では?」という問いが立ち、本研究が生まれた。依存方向は一方向 ——
**tehai が素材（実装と A/B 実測）を出し、本研究が一般化・定理化して引き返す**。
本研究は tehai のコードを継がない。

## 2. 共通 DNA — 外部錨で検証し、落ちた所だけ上位へ

本研究の中心機構（**検証ルーティング型エスカレーション**：配分を自己申告でなく
*実行検証*で決め、外部 gold で落ちたタスクだけ上位モデルへ回す）は、operator の
自己改善系が別々に到達していた同じ原理である。本研究はそれを**組織形態として定理化**した
（市場支配定理 `p* = w/s`：安いティアの完全解率 > コスト比なら市場が単一モデルを Pareto 支配）。

| 姉妹リポ | 本研究から見た位置 | 共有する原理（本研究での対応物） |
|---|---|---|
| **tehai / tehai-core** | 第 1 データ点・委譲の実装例 | 契約・予算・権限境界・Judge ＝ 統治膜 `m*` の実装 |
| **rinne / rinne-engine** | 生成層の自己検証 | 異種モデル床・決定的反例裁定 ＝「誤りの脱相関」「外部錨」 |
| **recurse / recurse-harness** | 横展開・promote/kill | 外部錨で測定 ＝ 検証ルーティング |
| **self-improve-arch** | 本番＋限界の決定 | 実テスト＋実ベンチ＋統計 ＝ measurement-first（§9） |
| **kyogi** | 合議オーケストレータ | 3-reviewer majority ＝ 監督スケーリング実証 |
| **cogsync** | 人間×AI サイクル同期 | 「薄い人間統治膜」の人間側 |
| **dialogic-mas-research** | 対話的マルチエージェント | 組織＝調整構造の別解（オープンダイアローグ背骨） |
| **KOKORO / jibun** | 人間主権・可読性の極 | 中心的緊張「最適性 vs 可読性」の*可読性*側 |

## 3. 中心的緊張の人間側

本研究の中心的緊張は **最適性（AI ネイティブ・流動・検証中心）vs 可読性（人間が統治・主権を保てる）**
（§5 / 統治膜 `m*`）。この *可読性* の極＝「人間が読めて統治できる薄い膜」を独立に
扱っているのが `KOKORO`（心理師の見立て→AI パーソナライズ）と `jibun`（自己理解）。
本研究の「薄い人間統治膜」と、KOKORO の「人間主権モデル」は同じ境界の表裏。

## 4. 含意

- 本研究の各実証（監督スケーリング・市場 3 レジーム・レース外部性・Goodhart）は、
  これら姉妹リポの**設計判断に還元できる** —— 例：`tehai` の Judge / risk-based review の
  最適な厚みは統治膜 `m*` の計測（`stakes` 依存）に対応する。
- 逆に姉妹リポ群は本研究の仮説の **live なテストベッド**である。
  `rinne`/`recurse`/`self-improve-arch` の運用実績は「検証ルーティング市場」の実地観測になる。
- → 一言：**soshiki-genron は、バラバラに作ってきた「AI を外部錨で律する」系プロジェクト群の
  理論的ハブ**。`tehai` がその第 1 実装、自己改善群が同型、`KOKORO` が人間統治膜の側。
