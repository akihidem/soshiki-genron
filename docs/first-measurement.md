# 最初の計測 — 通信コストと最適な組織構造

> 第一原理（[`method.md`](method.md)）の最初の具体化。概念（[`foundations.md`](foundations.md) F3）を**測れる仮説**へ落とし、決定的モデルで計測した。生の数値は [`../model/RESULTS.md`](../model/RESULTS.md)（再実行で再生成）。本ドキュメントは durable な解釈・位置づけ。

## 1. 問いを計測可能にする
- **概念（F3）**: 階層は「狭い通信帯域」への対処（管理者が O(n²) の通信経路を O(n) に funnel する装置, Galbraith）。通信が安い AI 域では階層は惰性化するはず。
- **理論（Malone 1987）**: IT が調整コストを下げると、構造は階層→市場へ寄る。
- **測れる形**: 「**通信コスト c_comm を高→低に動かすと、コスト最小の調整構造はどう変わるか**」。

## 2. モデル（[`../model/coordination.py`](../model/coordination.py)）
3つの純粋な調整戦略の総コストを、共通単位で第一原理から組む:

```
総コスト(構造, c_comm) = c_comm · 通信量 + 構造オーバーヘッド + 誤りコスト
```

| 構造 | 通信量が比例する量 | 構造OH | 誤り（検証様式） |
|---|---|---|---|
| flat（平ら/共有状態） | 相互依存の**辺 E** | なし | 外部検証→ノードで捕捉（安い rework） |
| hierarchy（階層） | **エージェント数 N**（E→N に funnel）＋残留ルーティング | 管理者OH（通信非依存） | 自部門レビューは盲点共有→**流出**（高い escape） |
| market（市場/競売） | 入札ラウンド × N | 取引OH | 外部＋最適割当で一部回避 |

各総コストは c_comm の**一次関数**なので交差点は解析的に厳密。

## 3. 計測結果（既定パラメタ）
- **flat↔hierarchy 交差点 c_comm\* ≈ 1.12**。同条件で通信がこれより高コストなら hierarchy、安いと flat が上回る。
- 3構造の勝者は c_comm を下げると **hierarchy →（中域）market → flat** と推移。**密度が強く効く**:
  - ρ=0.2（疎）: ほぼ全域で **flat**（辺が少なく直接調整が常に安い）。
  - ρ=0.4–0.6: 高 c_comm=**階層** / 中=**市場** / 極低=**平ら**。
  - ρ=0.8（密）: 階層→市場が支配、flat はほぼ出ない（密な辺で flat の通信が高すぎる）。
- **感度（仮定依存を明示）**: `mgr_overhead` 0→8 で交差点 0.61→2.65。**管理者オーバーヘッドが大きいほど階層の領域は狭い**。mgr_overhead=0 でも交差点 0.61 が残る（自部門レビューの盲点流出＋ルーティング税）。
- **検証軸の分離**: `same_error=True`（盲点流出ペナルティを中立化）でも交差点 c_comm\*≈0.51 が残る → flat 優位の一部は**純粋な調整コスト**由来で、検証（盲点流出）由来と分離できる。

## 4. 反証手段（測れる主張であることの担保）
- mgr_overhead>0 でも交差点が現れない／c_comm を下げても hierarchy が負けない → **F3 は偽**。
- market が低 c_comm 域で決して勝たない → 本モデル想定下で **Malone の市場化予測は偽**。
- 交差点が密度に依存しない → 「密度が効く」という主張は偽。

## 5. 位置づけ
- **F3 を支持**（条件付き）: 階層の優位は c_comm が高い時に限られ、通信が安くなると惰性化する —— ただし**管理者にオーバーヘッドがある**という経験的前提に懸かる（感度分析が明示）。
- **Malone を再現**: 調整コスト低下とともに階層→（市場）→平らへ。市場は中域、平らは極低域という二段の含意も得た。
- **tehai の A/B との関係**: tehai は実コードで2点を観測し flat 有利を得た（`tehai/experiments/`、private）。本モデルは解析的に曲線を出し、**同じ向き**（通信が安いと flat 有利）を**独立の経路**で示した —— 弱い相互裏取り。1点観測が曲線の低 c_comm 端に対応する。

## 6. 妥当性の限界
- 決定的な解析モデル。係数は第一原理的だが**仮定**。結論は質的主張（交差点の存在と向き）＋パラメタ依存（感度）として読む。単一数値を絶対視しない。
- 構造は3純粋型に限定（実組織は混合）。エージェント容量の有界性は未モデル（AI では緩いと仮定）。
- これは ② の**胚**であって完成形ではない。次は構造の混合・容量制約・実データ較正。

## 7. 次の計測（→ ②）
1. **容量制約**を入れる（エージェントの処理上限）。F1 分解の粒度がここで効くはず。
2. **統治膜の厚み**（F8）をパラメタ化し「人間介入の正答率・監査時間 vs 効率」のトレードオフ曲線を引く（foundations §5 の反証手段）。
3. 形式 backbone（Marschak & Radner のチーム理論）でこの cost 構造を裏取り／一般化（[`literature.md`](literature.md)）。
