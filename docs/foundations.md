# ③ 基礎 — 組織を機能から再導出する

> 方法は [`method.md`](method.md)。構造（チーム/役割/階層）から始めず、機能から始め、各機能の人間版が*なぜその形か*を拘束まで遡り、AI でその拘束が成立するかを問う。各節末に「② でどう反証されうるか」を付す。

## 0. 最小定義

> **組織**とは、*単一エージェントの限界を超える目標を、分割・再結合できるときに、それを担う構造と過程*である。

この定義は基層中立（人間でも AI でも成り立つ）。ここから「組織が必ず担う原始機能」を導く。構造（誰が誰の上司か）は定義に含まれない —— 構造は機能の一実装にすぎない。

## 1. 原始機能（どんな主体でも担う）

| # | 原始機能 | 問い |
|---|---|---|
| F1 | 目標の表現と分解 | 目標を扱える作業単位に変える |
| F2 | 割り当て | 作業を能力ある主体に結びつける |
| F3 | 相互依存の調整 | 順序・中間成果の受け渡し・衝突解消（誰が誰に何を要るか） |
| F4 | 情報の統合・伝播 | 部分成果と知識を一貫した全体に束ね、文脈を運ぶ |
| F5 | 品質保証・誤り訂正 | 欠陥を検出し直す（＝検証） |
| F6 | 適応・再計画 | 現実が計画から逸れたら組み替える |
| F7 | 動機づけ・整合 | 主体が*目標*を（自己利益でなく）追うようにする |
| F8 | 統治・説明責任 | 全体を可読に保ち、主人（principal）に答えられるようにする |

（F9 境界・同一性・継続は F8 に畳む。）

## 2. 拘束系譜と AI 判定（本研究の核）

各機能の人間版の形を、それを生んだ拘束まで遡り、AI で拘束が成立するかを判定する。

| 機能 | 人間版の形 ← 拘束（なぜそうなるか） | AIで拘束は | AIネイティブな形 |
|---|---|---|---|
| F1 分解 | 計画者/管理者。人の容量と在職で粒度が決まる | △ 縮小 | 検証/能力が要る粒度で動的・JIT分解。"人の職"は単位でない（**計測: [`../model/CAPACITY.md`] 容量↑で粒度↓＝AIは粗い分解**） |
| F2 割り当て | 固定役割・採用。専門家獲得が遅く高コスト | ✗ 逆転 | 役割はオンデマンド生成・揮発的。常駐しない |
| F3 調整 | 階層・会議。通信が遅く狭帯域 → 管理者が経路を削減 | ✗ 逆転 | 共有状態 / データフロー契約で調整。**帯域のための階層は惰性** |
| F4 統合 | 文書・引き継ぎ・暗黙知。記憶は私的・忘却・退職で消失 | ✗ 緩む | 共有・追記・永続メモリ / 黒板。引き継ぎ損失ゼロ |
| F5 検証 | QA部門・自部門レビュー（盲点共有）。高コストで選択的 | ⚠ 変質 | **検証が背骨**。全単位を独立/異種で検証＝外部錨 |
| F6 適応 | 遅い（再編は高コスト・政治的） | ✗ 緩む | 連続再計画。構造は流動 |
| F7 整合 | 巨大な管理装置。人は私利・サボり・離反しうる | ⚠ 変質して残る | "管理"が**仕様・検証・監督**へ変態（**計測: [`../model/ALIGNMENT.md`] 最適化圧 p\* は spec品質+検証で上限づき・超えると Goodhart で悪化**） |
| F8 統治 | 組織図・責任分担・監査。人間が結果に答える | ✅ 残る・むしろ重要 | AIネイティブ機構の上に**薄い人間統治膜** |

**② による反証手段**: 各行の「AI判定」は、② の実験基盤で「その拘束を on/off した構造」を同一課題に流し、機能の達成度・コスト・誤りで測れば PASS/FAIL が付く。例えば F3「階層は帯域のための惰性」は、フラットな共有状態構造と階層構造を、通信コストを変えながら比較すれば反証可能（tehai の A/B はその1点での観測：欠陥ゼロでも階層は約2.9×高コスト）。→ **計測済み（[`first-measurement.md`](first-measurement.md) / `../model/`）**: c_comm を掃引し flat↔hierarchy 交差点 c_comm\*≈1.12 を測定。高 c_comm=階層 / 安いと平ら、と F3 の向きを支持（ただし管理者OH>0 が前提＝感度分析で明示）。

## 3. 四分類 —— 何が惰性で、何が残るか

拘束系譜から、人間組織の機能は AI に対し4つに割れる:

- **惰性（vestigial）— AIには大方不要**: F2 固定役割、F3 帯域のための階層、F4 引き継ぎ/文書regime、F6 再編の遅さ。
  → これらを温存するのは擬人的負債。AIネイティブ構造はここを**平らに・オンデマンド・共有メモリ・連続再計画**にできる。
- **強く転移する**: 「**標準化**」（Mintzberg）—— 出力・スキル・規範の標準化は、人間でも AI でも有効な調整様式。AI では **契約・スキーマ・型・provenance** として一層強く効く（lossless に渡せるから）。
- **変質して残る（フロンティアその1）**: F5 検証・F7 整合（§4）。
- **そのまま残り、荷重部材（フロンティアその2）**: F8 統治・可読性（§5）。

## 4. フロンティア1 — 整合と検証は「消えず変質する」

最も非自明な点。人間組織の**管理層**は、突き詰めれば F7（動機の整合）と F5（品質の保証）のための装置だった。AI に私利は無い —— だが整合の問題は消えず、**別の問題に変態**する:

- 人間: 私利・怠慢・離反（opportunism） → 監督・評価・文化で抑える。
- AI: 報酬ハック・仕様ゲーミング・諂い（sycophancy）・分布ドリフト・確信的幻覚 → **仕様の精緻化・独立検証・監督**で抑える。

つまり「**管理（management）」は AI では「仕様＋検証＋監督（specification + verification + oversight）」へ変態する**。そして検証（F5）の性質も変わる:
- 人間の誤りは多くが*怠慢/能力* → 選択的レビューで足りる、しかも自部門レビューは盲点を共有する。
- AI の誤りは*確信的*に出る（沈黙しない誤りは捕まえにくい）。だが**独立・異種の検証は安い**。
- ゆえに AIネイティブでは、検証は一部門でなく**背骨**になり、全単位を独立/異種の検証器（外部錨）に通すのが合理。

> 系: AIネイティブの「管理層」は人を励ます層ではなく、**主張を独立に検証し仕様を締める層**。組織図の縦線は、検証のデータフローに置き換わる。

**② による反証**: 「自部門レビュー（同系統）」と「独立/異種検証」を同一欠陥分布に当て、見逃し率（特に確信的・盲点型欠陥）を比較。変質テーゼが偽なら差は出ない。

## 5. フロンティア2 — 統治と可読性は残り、荷重部材

F8 だけは AI で**緩まず、むしろ重要になる**。人間が AI に対し主権・説明責任を保つ必要があるからだ。ここが純効率設計を許さない:

- AIネイティブに最適化された構造（流動・検証中心・オンデマンド・市場的）は、**人間にとって読みにくく・統治しにくい**。
- 「チーム」「役割」「責任者」という人間組織の語彙は、効率には惰性でも、**統治の可読性**には荷重部材 —— 責任の所在と承認ゲートを人間に読ませる。

> 中心的緊張: **最適性（AIネイティブ）vs 可読性（人間統治）**。最適解はたぶん「AIネイティブな機構の上に、必要十分に薄い人間可読の統治膜を被せる」点にある。膜が厚すぎれば擬人的負債、薄すぎれば主権喪失。

**② による反証**: 統治膜の厚み（人間可読な構造をどれだけ残すか）をパラメタ化し、「人間の介入正答率・監査時間」対「効率」のトレードオフ曲線を引く。膜ゼロが常に最適なら本テーゼは偽。→ **計測済み（[`second-measurement.md`](second-measurement.md) / `../model/governance.py`）**: 内点最適 m\* が存在し stakes がしきい値を決める（<1.11 で m\*=0／>22.3 で m\*=1）。膜ゼロは常には最適でなく、§5 を**条件付き支持**。ただし**最大の脅威は多者レース**（[`ai-2027.md`](ai-2027.md) / `../model/race.py`）: 競争下の均衡膜は社会最適より薄くなる（race-to-the-bottom・AI-2027 の Race ending に対応）。**注: `oversight_error` を実測で接地（[`oversight-pilot.md`](oversight-pilot.md)）— capable な監督では ~0、巨大な能力差（opus 生成の巧妙欠陥 → e2b 監督）で 0.5 まで上昇。AI-2027 の →1（超人/neuralese）は測定可能モデルの外への外挿。併せて失敗は precision（過剰flag）でも現れ、能力とともに悪化（強い監督ほど厚い膜が良いとは限らない）。**

## 6. 暫定テーゼ

> AIネイティブの「組織」はたぶん**組織ではない** —— 型付き・証拠付き・記憶共有のデータフロー＋オンデマンドのエージェント生成、その上に薄い人間統治膜。「チーム/役割/階層」はその人間可読な投影にすぎない。

これは tehai の A/B 実測で*ある2点*として裏取りされたが、本研究はそれを*地面から*導いた。tehai を起点にしていない点が重要 —— 同じ結論に独立の経路で到達したことが、テーゼの頑健性の弱い証拠になる。
